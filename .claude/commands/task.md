# Команда /task — взять задачу в работу

## Назначение

Команда `/task` запускает рабочий цикл задачи: актуализирует контекст из `.claude/`,
выбирает задачу из backlog, выполняет её, обновляет `.claude/CLAUDE.md` и переносит
задачу в `done/` с отчётом.

---

## Структура директорий задач

```
E:\LocOll\Tasks\
├── backlog\          # задачи ожидающие выполнения (LL-NNN_slug.md)
├── done\             # выполненные задачи (с добавленным разделом ## ✅)
└── (корень)          # не используется — задачи живут только в backlog/ или done/
```

---

## Алгоритм выполнения /task

### Шаг 1 — Актуализация контекста

Прочитать все файлы из `.claude/`:
- `.claude/CLAUDE.md` — главный контекст: архитектура, стек, сервисы, порты, известные проблемы
- `.claude/commands/` — другие команды (для понимания системы)
- `server-access.md` в HomeLab если нужны детали подключения

Если в CLAUDE.md есть раздел `⚠️ Известные проблемы` — учесть их перед началом работы.

### Шаг 2 — Выбор задачи

Прочитать список файлов в `Tasks/backlog/`. Возможные сценарии:

**Backlog пуст** — сообщить: "Backlog пуст, новых задач нет. Добавь задачу в `Tasks/backlog/`."

**Одна задача** — взять её в работу автоматически, сообщить:
```
Беру в работу: LL-002 — DuckDB WASM загрузка данных, модальные окна
Читаю задачу...
```

**Несколько задач** — показать нумерованный список и спросить какую взять:
```
В backlog несколько задач:
1. LL-002 — DuckDB WASM загрузка данных, модальные окна
2. LL-003 — (название)
3. LL-004 — (название)

Какую задачу берём? (введи номер или ID)
```
Дождаться ответа пользователя, затем продолжить с выбранной задачей.

### Шаг 3 — Выполнение задачи

Прочитать файл задачи из `Tasks/backlog/`. Файл задачи содержит всё необходимое:
цель, контекст, конкретные шаги, критерии готовности.

Выполнять задачу строго по порядку выполнения из файла. После каждого крупного
шага — проверять критерий готовности для этого шага.

**Правила выполнения:**
- Подключение к серверу — только через paramiko (шаблон в CLAUDE.md)
- Если в ходе работы обнаруживается новая проблема не описанная в задаче — зафиксировать
  её в разделе "Обнаруженные проблемы" и продолжать, не останавливаться
- Если задача невыполнима из-за блокера — сообщить пользователю с описанием блокера

### Шаг 4 — Завершение задачи

После того как все критерии готовности выполнены:

**4a. Обновить CLAUDE.md**

Актуализировать разделы которые изменились в ходе выполнения задачи:
- Обновить статусы сервисов если что-то изменилось
- Добавить/убрать записи из раздела `⚠️ Известные проблемы`
- Обновить структуру проекта если добавились файлы
- Обновить реестр задач: перевести задачу из `🔲 в очереди` в `✅ выполнена`
- Обновить дату `*Последнее обновление*`

**4b. Добавить раздел завершения в файл задачи**

Дополнить файл задачи разделом перед `*Дата создания*`:

```markdown
---

## ✅ Статус: ВЫПОЛНЕНА

**Дата завершения:** YYYY-MM-DD

**Что сделано:**
- (краткое перечисление выполненного, 3-7 пунктов)

**Отклонения от плана:**
- (если были — описать; если не было — "Выполнено в соответствии с планом")

**Обнаруженные проблемы (перешли в следующие задачи):**
- (если есть; если нет — убрать этот пункт)
```

**4c. Переместить файл задачи**

Переместить файл из `Tasks/backlog/LL-NNN_slug.md` в `Tasks/done/LL-NNN_slug.md`.
Файл в `backlog/` удалить.

### Шаг 5 — Отчёт пользователю

Предоставить отчёт в следующем формате:

```
## ✅ Задача LL-NNN выполнена

**Задача:** (название)
**Время:** (примерное)

### Что сделано
(3-7 пунктов с конкретными изменениями)

### Проверка критериев
(таблица или список: критерий → результат проверки)

### Изменения в CLAUDE.md
(что обновлено)

### Следующие шаги
(если есть незакрытые вопросы или задачи которые стоит создать)
```

## Пример полного цикла

```
Пользователь: /task

Агент читает CLAUDE.md → видит известные проблемы (DuckDB blob: URL)
Агент читает Tasks/backlog/ → находит LL-002
Агент: "Беру в работу LL-002 — DuckDB WASM и модальные окна"

[выполняет шаги задачи]

Агент обновляет CLAUDE.md:
  - убирает DuckDB из Known Issues
  - обновляет реестр: LL-002 → ✅ выполнена

Агент дополняет LL-002_duckdb-fix-and-modals.md разделом ✅
Агент перемещает файл: backlog/ → done/

Агент: "## ✅ Задача LL-002 выполнена ..."
```

---

## Важные правила

Никогда не начинать выполнение не прочитав CLAUDE.md — там могут быть известные
проблемы или ограничения которые влияют на реализацию.

Если в ходе выполнения одной задачи находятся баги или улучшения выходящие за её
рамки — не делать их в рамках текущей задачи. Зафиксировать как "Обнаруженные
проблемы" и предложить создать новую задачу после завершения текущей.

Отчёт предоставлять всегда, даже если задача простая. Это главный артефакт
завершения рабочего цикла.
